# Pricing page layout template

Use with `uiux-layout-director` or `graphic-design-director` for SaaS/product monetization pages.

## Required structure

```text
01 value headline: who pays and why now
02 billing toggle: monthly/yearly, if relevant
03 plan cards: 2-4 plans max
04 recommended plan: one clear highlight, not all cards shouting
05 feature comparison: after plan cards, not before
06 trust/reassurance: refund/security/support/usage proof
07 FAQ: handle objections
08 final CTA
```

## Plan card rules

- Each plan needs a distinct user segment.
- Price must be visually close to billing unit.
- CTA labels should differ when intent differs: Start free, Contact sales, Upgrade.
- Avoid putting every feature in every card.
- Enterprise plan should explain the sales trigger.

## Visual rules

- Use one accent for recommended plan.
- Keep comparison tables scannable; freeze category labels on mobile if needed.
- Do not bury limitations in tiny low-contrast text.

## Open Design prompt add-on

```text
Create a pricing page with clear plan differentiation, one recommended plan, readable comparison, objection-handling FAQ, and honest upgrade paths. Avoid three identical pricing cards with random feature lists.
```
