# Landing page layout template

Use with `uiux-layout-director` for product/site homepages.

## Required structure

```text
01 hero: specific promise + product proof + primary CTA + secondary CTA
02 trust strip: logos, metrics, or deployment proof
03 problem frame: what current workflow costs users
04 product surface: screenshot/mockup/interactive artifact dominates
05 feature stack: 3-5 features grouped by user job, not by internal tech
06 workflow: before → during → after
07 social proof: quote, case, usage number, or realistic placeholder
08 pricing/packaging teaser if relevant
09 FAQ or objection handling
10 final CTA
```

## Layout rules

- Hero needs one visual surface, not floating decoration.
- Use asymmetric sections after the hero to avoid template sameness.
- Keep navigation to 5 top-level items max.
- Every section needs a job: persuade, explain, prove, or convert.
- Avoid generic feature cards unless each has product-specific detail.

## Open Design prompt add-on

```text
Generate a high-fidelity landing page. Use a strong product surface as the main visual anchor, asymmetric section rhythm, specific product copy, and clear conversion path. Avoid generic gradient SaaS slop and interchangeable icon-card feature grids.
```
