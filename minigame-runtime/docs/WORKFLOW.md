# MINIGAME 工作流

## 核心约束

1. **每次只做一个功能**  
   不允许在同一次修改中混入无关重构、视觉重做或新玩法。

2. **先设计，后开发**  
   复杂功能必须先写清楚设计目标、状态、操作、反馈和验收标准。

3. **Codex 是唯一代码执行者**  
   其他 AI 只做架构审查、任务拆解、中文内容生成和复盘，不同时改代码。

4. **所有修改必须可回滚**  
   每个阶段都要有清晰 Git commit，避免不可追踪的大改。

5. **禁止偏离主线**  
   当前唯一主线是异常系统模拟类小游戏，原型为“异常电梯控制台”。

## AI 角色分工

- Codex：写逻辑、接 UI、实现功能
- CC Switch / Claude：架构审查、任务拆解
- Qwen：中文剧情、规则、日志文本生成
- DeepSeek：低成本事件/文案草稿批量生成
- Hermes/Hammers：系统记忆、决策记录、复盘
- ChatGPT：总产品决策、方向控制

## 单次开发流程

1. 明确本次唯一目标
2. 如有必要，更新设计文档
3. 写测试或可验证验收点
4. 实现最小可用功能
5. 本地运行验证
6. Git 提交
7. 如需远程同步，推送到 `main`

## 禁止事项

- 不允许无任务重构
- 不允许同时做多个玩法方向
- 不允许把项目改回普通小游戏合集
- 不允许为了视觉效果牺牲状态系统、事件系统、反馈系统和广告循环
