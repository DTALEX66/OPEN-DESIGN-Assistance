# P1 — 变现循环扩展：隐藏日志 + 假结局

## 一、设计目标

在现有"失败→广告复活"基础上，增加两个新的广告变现触点：
1. **隐藏日志** — 异常事件会留下加密日志片段，需广告解锁才能阅读完整内容
2. **假结局** — 玩家连续失败 N 次后触发特殊结局画面，需广告揭示真相

核心原则：
- **不改变现有核心玩法** — 只是锦上添花的内容解锁
- **广告点位自然融入** — 玩家想看"到底是什么"时主动选择观看
- **可配置** — 所有参数放在 `gameConfig.js`

---

## 二、隐藏日志系统

### 数据设计

```js
// 状态新增字段
state.hiddenLogs = [
  {
    id: 'phantom_floor_log',
    title: '第13层施工记录',
    locked: true,
    // 解锁后显示的完整内容
    content: '2019年施工记录：第13层在竣工前被从建筑图纸中删除。\n原因：施工期间发生Ⅲ级安全事件，3名工人失踪。',
  },
  // ...
]

state.adHintsUsed: 0  // 广告解锁消耗（不计入复活次数）
```

### 隐藏日志来源

每个异常事件可以关联一条隐藏日志。当该异常首次触发时，对应的隐藏日志添加到 `state.hiddenLogs`。

初始隐藏日志（与现有 12 种异常事件配对）：

| 异常 ID | 隐藏日志标题 | 内容片段 |
|---|---|---|
| `phantom_floor` | 第13层施工记录 | 2019年施工期间发生Ⅲ级安全事件，3名工人失踪 |
| `zero_passenger_shadow` | 乘客记录异常说明 | 乘客传感器在非营业时段检测到热源信号 |
| `stop_failure` | 急停系统维护日志 | 急停回路曾在维护中被标记为"不可用" |
| `negative_floor` | 地下层勘测报告 | 建筑实际有地下4层，但图纸只标注了B1-B2 |
| `power_drain` | 备用电源异常报告 | 备用电源曾被非授权设备分接 |
| `log_echo` | 日志系统诊断 | 重复输出可能来自上一值班员的求救信号 |
| `camera_delay` | 监控系统校准记录 | 延迟与第13层信号干扰有关 |
| `auto_button` | 控制系统审计 | 自动按钮信号追踪到已停用的5号服务器 |
| `door_refuse` | 门控系统事故报告 | 门控模块在3次异常重启后显示"拒绝执行" |
| `weight_mismatch` | 传感器校验记录 | 0kg读数与红外检测矛盾——红外曾捕获到移动 |
| `floor_jump` | 楼层定位日志 | GPS楼层定位模块在跳跃前后记录不一致 |
| `emergency_lights` | 应急照明测试报告 | 应急灯在无触发信号下启动——供电线路存在寄生回路 |

### 交互流程

1. 异常事件触发 → 关联隐藏日志加入 `state.hiddenLogs`（如果还没加过）
2. 玩家点击"查看日志" → 显示常规日志 + "有 N 条加密记录可解锁"
3. 玩家点击"广告解锁隐藏日志" → 模拟广告 → `locked: false` → 完整内容显示
4. 隐藏日志内容显示在监控区域或专门的弹窗

### 广告消耗类型区分

```
state.adRevivesUsed  — 复活次数（已有）
state.adHintsUsed    — 隐藏日志解锁次数（新增）
```

两者在 UI 上分别显示，但共享同一个"模拟广告"机制。

## 三、假结局系统

### 触发条件

```js
// gameConfig 新增
fakeEnding: {
  consecutiveFailuresThreshold: 5,  // 连续失败N次触发
  cooldownFailures: 3,              // 触发后需再失败N次才能再看
}
```

**连续失败定义**：失败后**没有成功值守到倒计时结束**就直接再次失败。
- 每次成功值守到倒计时结束（remaining <= 0 → gameOver with success）重置计数
- 触发假结局后计数重置

### 交互流程

1. 玩家连续失败 N 次
2. 下一次失败时 → 不显示普通失败弹窗，而是显示假结局画面
3. 假结局内容（文字 > 系统日志风格）：
   > "系统检测到操作员第 5 次系统崩溃。\n
   > 根据《异常控制员守则》第 7 条，您已被标记为"异常关联人员"。\n
   > 前 4 次记录已被永久删除。\n
   > 建议您立即离开控制台并联系安保部门。"
4. 提供两个选项：
   - "重新开始"（普通重启）
   - "观看广告揭示真相" → 广告后显示真实背景故事
5. 真相内容（广告后解锁）：
   > "这不是第一次，也不会是最后一次。\n
   > 这座建筑的异常系统从未被修复。\n
   > 每一任值班员最后都变成了「异常事件」本身。\n
   > 系统日志中关于「乘客」的记载——都是前任值班员的热源信号。\n
   > 你现在坐的位置，就是上一任值班员被发现的地方。"

### UI 设计

假结局弹窗复用现有的 `failureOverlay`，但样式更特殊（红色调更重、文字闪烁、带扫描线故障效果）

## 四、配置文件变更

```js
// gameConfig.js 新增段
adContent: {
  adVideoDuration: 3000,     // 模拟广告时长（毫秒）
},

hiddenLogs: {
  maxUnlockedPerRun: 3,      // 每局最多解锁数
},

fakeEnding: {
  consecutiveFailuresThreshold: 5,
  cooldownFailures: 3,
},
```

## 五、验收标准

- [ ] 异常事件触发后，关联隐藏日志出现在列表中
- [ ] 点击"查看日志"可看到加密记录的提示
- [ ] 点击广告解锁后可阅读完整隐藏内容
- [ ] 连续失败 N 次后触发假结局画面
- [ ] 假结局后广告解锁真相内容
- [ ] 假结局计数在成功值守后重置
- [ ] 所有新增数据在复活回滚时正确处理
