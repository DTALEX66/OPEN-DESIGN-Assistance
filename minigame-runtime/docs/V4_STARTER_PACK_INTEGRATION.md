# V4 Starter Pack Integration

本文件记录 `MINIGAME_V4_Starter_Pack.zip` 已被吸收到当前项目的方式。

## 原则

- 不覆盖当前项目代码。
- 不解压覆盖已有目录。
- 不把 zip 当作源码提交。
- 只吸收其中的产品策略、内容包协议、底座评估方式、营销模板和试玩检查标准。

## 吸收后的项目定位

当前项目继续保持：

```text
单人 AI 驱动的小游戏生产与变现系统
```

不是单个小游戏，而是一条可复制生产线：

```text
异常系统原型 → 内容包 / 皮肤 → H5 / 微信 / 抖音适配 → IAA 广告循环 → 小红书 / 抖音引流 → 数据复盘 → 换皮复制
```

## V4 Starter Pack 中被采纳的部分

### 1. 产品主线

采纳：

- Phaser/H5 或当前 H5 原型用于快速验证。
- Cocos Creator 作为商业上线候选底座。
- Godot / VN / WebGAL 作为叙事结构参考，不作为当前第一桶金主线。
- 第一内容方向保持 `monitor_anomaly` / 异常监控室。

不采纳：

- 不从零重建项目目录。
- 不覆盖当前源码、测试、构建和皮肤系统。
- 不下载大量候选仓库，除非有明确小任务。

### 2. 内容包协议

采纳 `content-pack.json` 作为未来换皮 / 题材复制的数据接口参考，详见：

```text
docs/CONTENT_PACK_SPEC.md
```

当前项目已有 `src/skins/*/skin.json`，短期内不强行迁移；后续应逐步建立映射：

```text
content-pack.json → skin.json / anomalies / hiddenLogs / adSlots / shareText
```

### 3. UI / 设计方向

采纳 Starter Pack 的 UI 定位：

```text
异常监控系统 / 夜班值守终端 / 控制台界面
```

这与当前 YOLO 边界一致：

```text
暗色异常控制台 / HUD / CCTV / 终端日志氛围
```

所有 UI 改动继续必须浏览器视觉验收，不能像表单、后台管理页或调查问卷。

### 4. IAA 广告点

采纳以下广告位作为当前系统的长期广告循环：

- 失败复活
- 隐藏线索 / 加密记录解锁
- 假结局提示 / 真相揭示
- 解锁下一段监控

第一阶段只做模拟广告和占位，不接真实广告 SDK。

### 5. 营销与数据闭环

采纳小红书 / 抖音测试方向：

- 小红书：监控找异常、悬疑互动标题、评论区互动。
- 抖音：15 秒找异常过程、失败反转、复活钩子。

后续应在 `docs/` 或 `marketing/` 中沉淀：

- 发布标题
- 15 秒脚本
- 截图点位
- 数据记录表
- 是否继续 / 砍掉 / 换皮标准

## 和当前项目的关系

当前项目已经具备 Starter Pack 目标中的一部分实现：

- H5 原型
- 皮肤系统
- 异常事件
- 隐藏日志
- 模拟广告复活
- 假结局
- 微信构建
- 自动化测试
- 浏览器预览验收标准

因此 V4 Starter Pack 不是替代品，而是：

```text
产品 / 内容 / 工作流蓝图
```

## 后续吸收任务建议

1. 建立 `content-pack.json` 到 `skin.json` 的字段映射测试。
2. 把 `monitor_anomaly` 示例事件转成当前 anomaly skin 数据格式。
3. 增加营销素材文档：小红书标题、抖音 15 秒脚本、截图需求。
4. 建立 playtest checklist，并在每轮 UI 验收后记录是否通过。
5. 建立底座候选评分表，但不下载大量仓库。
