# 异常电梯控制台 — 游戏设计总纲

## 一句话定义

玩家操作一台不断出错的电梯控制台，在异常扩散前维持系统运行，并通过失败复活、提示解锁和隐藏日志形成 IAA 广告循环。

## 玩家身份

夜班电梯异常控制员。玩家不是英雄，也不是侦探，而是被要求“按流程处理异常”的值班操作员。

## 核心体验

> 玩家不是在看故事，而是在操作一个不断出错的系统。

体验关键词：

- 控制台 UI
- 电梯/监控/系统状态
- 异常事件触发
- 操作反馈
- 失败与复活循环

## 核心系统

### 1. State Machine

维护电梯与系统状态：

- 当前楼层
- 门状态
- 移动状态
- 电源
- 稳定度
- 异常等级
- 乘客数
- 游戏是否结束

### 2. Action System

玩家通过按钮执行操作：

- 开门
- 关门
- 上行
- 下行
- 急停
- 系统重启
- 查看日志

每次操作必须产生明确反馈。

### 3. Event System

系统根据时间、状态和玩家操作触发异常事件，例如：

- 电梯停在不存在的楼层
- 监控延迟
- 门外有人但乘客数为 0
- 急停失效
- 楼层显示为负数

### 4. Feedback System

通过以下方式反馈系统变化：

- 系统日志
- 警告提示
- 监控画面文字
- 屏幕闪烁
- 失败提示

### 5. Ad Loop System

第一阶段先用模拟广告实现：

- 失败 → 模拟观看广告复活
- 连续失败 → 模拟广告提示
- 隐藏日志 → 模拟广告解锁

## 最小可玩版本验收标准

玩家可以完成一局 60 秒模拟：

1. 看到控制台状态
2. 点击操作按钮改变电梯状态
3. 触发至少 5 种异常事件
4. 操作错误会提高异常等级或导致失败
5. 失败后出现“观看广告复活”按钮
6. 点击后恢复到可继续游玩的状态
